# Generate Operations Report

This skill generates a daily operations report (markdown format) containing merged pull requests and closed issues from the current git repository for "today" (local timezone).

## Workflow

1. **Identify the repository**: Get the current git repository's owner and name
2. **Determine "today"**: Use the local timezone to determine today's date range
3. **Fetch merged PRs**: Get all PRs merged today using the GitHub MCP server
4. **Fetch closed issues**: Get all issues closed today using the GitHub MCP server
5. **Generate report**: Create a well-formatted markdown report
6. **Save to file**: Save the report to the current directory with a timestamped filename

## Step-by-step Instructions

### Step 1: Get repository information

Use the `Bash` tool to get the current repository's GitHub owner and name:

```bash
git remote get-url origin 2>/dev/null | sed -E 's/.*github\.com[:\/]([^\/]+)\/([^\/]+)(\.git)?$/\1 \2/'
```

This will output: `<owner> <repo>`

If this fails (not a git repo or no origin remote), inform the user and exit.

### Step 2: Determine date range for "today"

Use local timezone to determine today's start and end timestamps in ISO 8601 format.

You can use a bash command to get:
- Start of today: midnight local time, converted to ISO 8601 UTC
- End of today: 23:59:59 local time, converted to ISO 8601 UTC

Example approach:
```bash
# Get start of today in local timezone, convert to UTC ISO format
date -v0H -v0M -v0S +"%Y-%m-%dT%H:%M:%S" 2>/dev/null || date -d "today 00:00:00" +"%Y-%m-%dT%H:%M:%S"
```

### Step 3: Fetch merged pull requests

Use the `mcp__github__search_pull_requests` tool with:
- Query: `is:pr is:merged repo:<owner>/<repo> merged:>=<start-of-day>`
- Sort: `updated`
- Order: `desc`

### Step 4: Fetch closed issues

Use the `mcp__github__search_issues` tool with:
- Query: `is:issue is:closed repo:<owner>/<repo> closed:>=<start-of-day>`
- Sort: `updated`
- Order: `desc`

### Step 5: Generate the markdown report

Create a well-structured markdown report with the following sections:

```markdown
# Daily Operations Report - <Date>

**Repository:** `<owner>/<repo>`
**Report Generated:** <timestamp>

## Summary

| Metric | Count |
|--------|-------|
| Merged PRs | <count> |
| Closed Issues | <count> |
| **Total Activity** | **<total>** |

## Merged Pull Requests

### #<number>: <title>
- **Author:** @<author>
- **Merged:** <time>
- **URL:** <url>
- **Description:** <description preview>

### #<number>: <title>
...

## Closed Issues

### #<number>: <title>
- **Author:** @<author>
- **Closed:** <time>
- **URL:** <url>
- **Labels:** <labels>
- **Description:** <description preview>

### #<number>: <title>
...

---
*Report generated by Claude Code*
```

For descriptions, include up to 3 lines. If longer, truncate with "..."

### Step 6: Save the report

Save the report to the current directory with the filename:
`ops-report-<YYYY-MM-DD>.md`

Use the `Write` tool to create the file.

## Edge Cases

- **No activity**: If no PRs were merged and no issues were closed, still generate the report with "No activity recorded" messages.
- **Not a git repo**: Inform the user and exit gracefully.
- **No GitHub remote**: If the origin is not GitHub, inform the user this skill requires a GitHub repository.
- **API errors**: If the GitHub MCP calls fail, report the error and suggest checking credentials.
